The pipeline runs five steps, top to bottom. Each produces a named dataframe.

## Pipeline Orchestration

`requisition_iq.py` runs the five steps in sequence:

```python
# requisition_iq.py

import data_importer as dimp
import iq

# ------------------------------ LOAD CONFIG
config = dimp.get_dataframe_config("filled_req_iq_cluster_1_df")

observations_df = config["observations_df"]
population_df = config["population_df"]
schema = config["schema"]
unique_id = config["unique_id"]
global_metrics_selected = config["global_metrics_selected"]
cluster_list_of_lists = config["cluster_list_of_lists"]
sort_by_for_top_n = config["sort_by_for_top_n"]

# ------------------------------ #1: BUILD OBSERVATION DATAFRAME
metric_config_observations_df = iq.step_1_build_observation_df(
    observations_df, global_metrics_selected, schema
)

# ------------------------------ #2: BUILD SAMPLE POPULATION DATAFRAMES
sample_population_groupby_lineage_unique_id_mean_sd_df, \
sample_population_groupby_lineage_unique_id_survival_df = iq.step_2_build_sample_population_dfs(
    population_df, global_metrics_selected, cluster_list_of_lists,
    unique_id, schema, sort_by_for_top_n
)

# ------------------------------ #3: BUILD OBSERVATIONS WITH ALL POSSIBLE LINEAGE_UNIQUE_ID's
observations_with_all_possible_lineage_unique_ids_df = iq.step_3_build_observations_with_all_lineage_ids(
    observations_df, global_metrics_selected, cluster_list_of_lists, unique_id, schema
)

# ------------------------------ #4: ADD Z-SCORES TO OBSERVATIONS BASED ON SAMPLE POPULATION GROUPS
observations_with_sample_population_z_scores_df = iq.step_4_add_z_scores_from_sample_population(
    observations_with_all_possible_lineage_unique_ids_df,
    sample_population_groupby_lineage_unique_id_mean_sd_df, unique_id
)

# ------------------------------ #5: ADD SURVIVAL PERCENTILES TO OBSERVATIONS BASED ON SAMPLE POPULATION GROUPS
observations_with_sample_population_group_survival_df = iq.step_5_add_survival_from_sample_population(
    observations_with_all_possible_lineage_unique_ids_df,
    sample_population_groupby_lineage_unique_id_survival_df, unique_id
)
```

The configuration loaded at the top resolves through `dataframe.yml` to produce all of the inputs the steps need. See [Schemas & Configs](/descriptive-systems/schemas-configs) for the YAML driving this.

## Step 1: Build the Observation Dataframe

Current quarter reqs tagged with the metrics defined in the schema.

The observation set is whatever the `dataframe.yml` config defines as the current observation period. For the default config, that's `calendar_quarter = "2025 Q4"`.

```yaml
# dataframe.yml
filled_req_iq_cluster_1_df:
  data_source: filled_reqs
  observations:
    column: calendar_quarter
    equals: "2025 Q4"
```

Each requisition in this set is tagged with its schema-defined analytical metrics — `applicants_total`, `time_to_fill`, `percent_reviewed`, `days_to_first_screen`, etc.

## Step 2: Build Sample Population Dataframes

Prior quarter fills grouped by context lens. Each context lens has a configurable `top_n` sample threshold.

```yaml
# dataframe.yml
  population:
    column: calendar_quarter
    not_equals: "2025 Q4"
```

The population is everything *not* in the observation period. Historic completed reqs that form the comparison baseline.

The engine groups the population data by context lens (see [How Context Works](/descriptive-systems/how-context-works) for the lens hierarchy). Each grouping must meet the `top_n` threshold to qualify as a valid context lens.

The grouping logic is in `etl_functions.py`:

```python
def new_group_by_top_n_columns_df(
        df: pd.DataFrame,
        on_columns: list[str],
        sort_column: str,
        top_n: int,
        ascending_equals: bool,
        unique_id: str
) -> pd.DataFrame:

    # Filter to groups meeting top_n threshold, then sample top_n records per group
    df = df.sort_values(by=sort_column, ascending=ascending_equals)
    df["count"] = df.groupby(on_columns)[unique_id].transform("count")

    if top_n is not None:
        df = df[df["count"] >= top_n]
        df["cumulative_count"] = df.groupby(on_columns)[unique_id].transform("cumcount")
        df = df[df["cumulative_count"] < top_n]

    return df
```

Groups that don't meet `top_n` get filtered out. The engine moves to the next broader grouping.

## Step 3: Expand Observations Across All Possible Lineage IDs

Each observation requisition gets multiplied across every context lens that qualifies for it.

A single requisition might end up represented seven times in the expanded dataframe — once per qualifying context lens. The `lineage_unique_id` column tracks which lens each row corresponds to.

The lineage column construction is in `etl_functions.py`:

```python
def add_lineage_columns(df,
                        lineage_cluster_name,
                        lineage_cluster_group,
                        lineage_columns,
                        lineage_column_level,
                        sort_column=None,
                        top_n=None,
                        lineage_target_column=None) -> pd.DataFrame:

    if df.empty:
        df["lineage_unique_id"] = None
    else:
        df["lineage_cluster_name"] = lineage_cluster_name
        df["lineage_unique_id"] = lineage_cluster_group + "_" + \
            df[lineage_columns].astype(str).agg("_".join, axis=1)
        df["lineage_cluster_group"] = lineage_cluster_group
        df["lineage_cluster_group_level"] = lineage_column_level
        df["lineage_column_level"] = len(lineage_columns)
        df["lineage_target_column"] = lineage_target_column
        df["lineage_metric_value"] = df[lineage_target_column]
        df["lineage_dict_name"] = f"{lineage_cluster_group}_{lineage_columns}"

    df["lineage_timestamp"] = pd.Timestamp.now()
    return df
```

The `lineage_unique_id` becomes the key that joins observations to their sample population groups in subsequent steps.

## Step 4: Z-Score Each Observation

For each observation row (req × context lens), compute the z-score against the sample population group's mean and standard deviation.

```python
def add_z_score_to_df(df, target_column, mean_name="mean", sd_name="sd", column_name="z_score"):
    # Standardizes target_column relative to group mean/sd
    df[column_name] = np.where(
        df[sd_name] == 0, 0,
        (df[target_column] - df[mean_name]) / df[sd_name]
    )
    return df
```

A z-score of `-1.48` against `company_Tech_Engineering_Entry` means the requisition's metric value is 1.48 standard deviations below the mean of that peer group.

The schema defines the direction that matters. For `time_to_fill`, lower is better (`direction: -1`). For `percent_reviewed`, higher is better (`direction: 1`). The signal interpretation accounts for direction.

## Step 5: Add Survival Percentiles

The cumulative distribution position of each observation within its sample population group.

```python
def new_lineage_df_group_by_cumulative_count_running_percent(df, target_column, lineage_unique_id):
    # Builds distribution across target_column values with cumulative percent within each group
    if df["lineage_top_n"].notna().any():
        top_n_map = df.groupby(lineage_unique_id)["lineage_top_n"].first()
    else:
        top_n_map = df.groupby(lineage_unique_id)[target_column].count()

    df = df.groupby(lineage_unique_id)[target_column].value_counts().reset_index(name="count")
    df = df.sort_values(by=[lineage_unique_id, target_column], ascending=[True, True]).reset_index(drop=True)
    df["cum_count"] = df.groupby([lineage_unique_id], observed=True)["count"].cumsum()

    df["top_n"] = df[lineage_unique_id].map(top_n_map)
    df["percent"] = df["cum_count"] / df["top_n"]
    return df
```

Survival percentiles bucket into quartiles:

```python
def classify_lineage_df_survival_quarter_bins(df, percent_column, lineage_target_column):
    df["bin_survival"] = pd.cut(
        df[percent_column],
        bins=[0, 0.25, 0.5, 0.75, 1],
        labels=["bin_survival_25th", "bin_survival_50th",
                "bin_survival_75th", "bin_survival_100th"]
    )
    return df
```

Reader sees both signals on each row — the z-score (how far from the group's mean) and the survival percentile (where in the group's distribution this req falls).

## Pipeline Output

Five dataframes. Each named. Each inspectable.

- `metric_config_observations_df` — observations with schema metrics
- `sample_population_groupby_lineage_unique_id_mean_sd_df` — peer group statistics
- `sample_population_groupby_lineage_unique_id_survival_df` — peer group survival distributions
- `observations_with_sample_population_z_scores_df` — observations with z-scores per context lens
- `observations_with_sample_population_group_survival_df` — observations with survival percentiles per context lens

Reader can audit any of these at any point in the pipeline. Nothing hidden. Nothing assumed.
